样式初始化原理

  • 为什么要样式初始化？

    通常我们所看到的页面的字体大小、字体颜色、背景色、图片等都可以通过我们编写css来控制标签的样式；然而，如果没有初始化样式时，我们的页面会在不同的浏览器中表现不一致；尤其在移动端开发，比较棘手；于是前辈们提出，统一初始化样式，然后再去编写样式，这样完美解决了样式不一致的表现问题。其原理可以说就是在请求我们的样式之前，放一个统一样式文件，引入它就可以了；如大家熟知的common.css、base.css等；

移动端适配推荐

  • window.js

    默认1rem=75px;，需要修改请自己修改

  • vue脚手架适配组件postcss-pxtorem

    可以把px转成rem;使用方法如下:
    $ npm install postcss-pxtorem -D
    2、修改根目录 .postcssrc.js
    文件 找到 plugins 属性新增pxtorem的设置 "postcss-pxtorem": { "rootValue": 32, "propList": ["*"] }
    按照上述配置项目后，即可在开发中直接使用 px 单位开发。
    例如设计给出的设计图是 750 * 1136，那么可以直接在页面中写
    body { width: 750px; height: 1136px; } 将被转换为 body { widht: 23.4375rem; height: 35.5rem; }
    注意：此方法支持import 和 .vue单文件中style。暂不支持style中使用@import url();
